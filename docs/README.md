等。

## 文档规范
1. 所有文档应当清晰标注其用途和适用范围
2. 代码示例应当注明是否可直接使用
3. 文档应当及时更新以反映最新的项目状态
4. 推荐使用 Markdown 格式编写文档
5. 对于包含代码的文档，使用适当的文件扩展名（如 .ts, .md 等）
